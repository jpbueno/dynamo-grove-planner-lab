💡 Why This Matters for LLM Serving

🚫 Without Gang Scheduling

K8s schedules pods individually:

  • 3/4 components land, 4th has no GPU
  • Idle GPUs, broken pipeline, manual fix

Grove PodGangs: all-or-nothing scheduling.

🚫 Without SLA-Aware Scaling

HPA watches CPU/memory, not LLM metrics:

  • TTFT and ITL are what matter
  • HPA can't derive GPU count from latency

Planner uses profiling data every cycle.

🌐 Without Topology Awareness

K8s places pods wherever resources fit:

  • KV cache over Ethernet instead of NVLink
  • Both replicas on same node = zero HA

Grove packs within NVLink, spreads across nodes.

🔢 Without Startup Ordering

K8s starts all pods simultaneously:

  • Workers register before frontend = crash loop
  • Decode before prefill = connection refused

Grove startsAfter: frontend → workers → planner.

🌐 Topology Rules

topologyConstraint.packDomain

Within a Replica — Packed

Components placed within the same NVLink/NVSwitch domain (same node or same GB200 NVL72 tray) for max bandwidth KV cache transfers.

PodCliqueSet replicas > 1

Across Replicas — Spread

Full pipeline replicas spread across different nodes or racks for HA. Cross-rack traffic uses InfiniBand or RoCEv2 over Ethernet.

📌
Where you set topology

Add topologyConstraint at PodCliqueSet, ScalingGroup, or PodClique level in your YAML. Reference a ClusterTopology by name and set packDomain. Default: no topology constraint — K8s schedules wherever it fits (the problem we’re solving).

🏗 ClusterTopology: Portable Topology Definitions

Admins define topology as a CRD. Workloads reference by name -- portable across clouds.

Admin Defines: ClusterTopology
apiVersion: grove.io/v1alpha1
kind: ClusterTopology
metadata:
  name: h100-topology
spec:
  levels:
    - domain: zone
      key: topology.kubernetes.io/zone
    - domain: rack
      key: kubernetes.io/rack
    - domain: host
      key: kubernetes.io/hostname
Workload References It
topologyConstraint:
  topologyName: h100-topology
  packDomain: "rack"

Same workload YAML works on any cluster — just swap the topology reference.

3 levels: PodCliqueSet, ScalingGroup, PodClique. Children can't be broader than parent.

🔌 Multi-Node NVLink (MNNVL)

What It Is

  • NVLink across nodes within the same rack/tray (e.g., GB200 NVL72 = 72 GPUs via NVLink)
  • Each replica gets its own ComputeDomain + IMEX channel
  • Cross-rack uses InfiniBand or RoCEv2 — not NVLink

Key Constraints

  • One IMEX channel per node — MNNVL workloads can’t share nodes
  • Homogeneous GPUs only — no mixing A100 + H100
  • Status delayed — active only after pods are scheduled

⚡ Advanced Planner Capabilities

🛡 Graceful Scale-Down

Zero dropped requests during scale-down:

  1. Stop sending new requests to the target worker
  2. Wait for all in-flight token generation to complete
  3. Drain the KV cache entries for active sequences
  4. Terminate the pod and release the GPU

terminationDelay defaults to 4h for long-running sequences.

📊 Metrics Beyond TTFT/ITL

Four signal categories from Prometheus:

TTFT / ITL Latency SLA compliance
KV Cache Load Memory pressure on decode workers
Prefill Queue Backlog depth on prefill workers
Request Rate Throughput for load prediction

Anticipate bottlenecks before SLA breach.

🔄 Dynamic Serving Mode

Dynamically switches disaggregated/aggregated serving per request.

Decision factors:

  • KV cache transfer cost — aggregated if transfer > single-GPU time
  • Queue wait time — shift idle decode GPUs to prefill
  • SLO headroom — fewer GPUs when latency has margin

Adapts not just how many GPUs but how they're used.

📚 Glossary

Latency

TTFTTime To First Token. Measures how long before the model starts generating output after receiving a prompt.
ITLInter-Token Latency. The time between consecutive output tokens during the decode phase.
P9999th percentile. "99% of requests complete within this latency." The standard SLA measurement.
SLAService Level Agreement. The performance contract (e.g., "P99 TTFT < 50ms").

Architecture

PrefillThe compute-bound phase that processes the entire input prompt and builds the KV Cache.
DecodeThe memory-bound phase that generates tokens one-at-a-time using the KV Cache.
KV CacheKey-Value cache. Stores intermediate computations from prefill so decode doesn't recompute them.
Disaggregated InferenceArchitecture pattern that separates prefill and decode onto different GPU pools for independent scaling.

Planner

DGDRDynamoGraphDeploymentRequest. CRD containing model profiling data and SLA targets.
DGDDynamoGraphDeployment. CRD containing the concrete deployment spec with replica counts.
Correction FactorRatio of observed latency to SLA target. >1.0 means scale up, <1.0 means scale down.
Load PredictorAlgorithm that forecasts future traffic (constant, ARIMA, Kalman, Prophet).

Grove / Kubernetes

PodCliqueSetCRD that defines a multi-component inference topology as a single declarative unit.
PodCliqueA single role/pod in the inference pipeline (e.g., one prefill worker).
ScalingGroupA set of identical PodCliques that scale as a unit.
PodGangOne complete replica of all components, scheduled atomically (all-or-nothing).
Gang SchedulingEnsures all pods in a group are scheduled together or not at all.
Topology-Aware PlacementScheduling that respects GPU interconnect topology (NVLink, NVSwitch) and failure domains.
