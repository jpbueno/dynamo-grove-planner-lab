Seeing It Live

Walking through the actual demo — from YAML manifests to Grafana dashboards, watching Grove and the Planner scale a Qwen model on A100 GPUs in real time.

The Demo Setup

💻
Like a fire drill

A fire drill simulates a real emergency in a controlled setting. This demo simulates production traffic hitting a real Dynamo deployment — same components, same scaling logic, just compressed to 2.5 minutes so you can watch it happen.

The demo deploys Qwen3-0.6B on A100 GPUs with these components:

🌐

Frontend

Receives requests via OpenAI-compatible /v1/completions endpoint. Exposes Prometheus metrics.

⚡

Prefill Worker

Processes prompts. Runs vLLM with enforce-eager mode for fast model loading in the demo.

🔄

Decode Worker(s)

Generates tokens. Starts at 1, scales to 2 under load, back to 1 when traffic drops.

📊

SLA Planner

Checks Prometheus every 20s. Targets: TTFT < 50ms, ITL < 10ms. Scales decode when breached.

GPU time-slicing (2 replicas per physical GPU) allows the scaled-up decode worker to share a GPU with the existing one.

What the Grafana Dashboard Shows

The demo uses a pre-built Grafana dashboard with four panels:

The dashboard also shows ITL (same pattern), request rate, and inflight requests. When the TTFT or ITL line crosses above the red SLA target line, the Planner scales up.

Demo Timeline: Scale Up, Then Scale Down

1
Heavy Load (90s, 15 req/s)

Load generator fires 15 requests per second. Latency climbs. Within 20s, the Planner detects ITL exceeding 10ms and patches the DGD: VllmDecodeWorker replicas: 1 → 2.

2
New Pod Appears

Grove creates the new decode pod. KAI Scheduler places it on the same node via GPU time-slicing. The model loads, pod becomes Running (1/1), and starts serving traffic.

3
Cool Down (60s, 1 req/s)

Traffic drops to a trickle. The Planner sees low utilization and scales decode back to 1. The extra worker is drained and its GPU is released.

The Three Screens to Watch

During the demo, three views run simultaneously in a tmux session plus a browser:

📊
Grafana (browser)

TTFT and ITL graphs with red SLA target lines. When the line crosses above the target, the Planner will react.

📜
Planner Logs (tmux top)

Watch for: Updating decode component VllmDecodeWorker to desired replica count 2 — this is the Planner making its scaling decision.

📦
k9s Pods (tmux bottom)

A new decode pod appears as ContainerCreating, transitions to Running (1/1), and eventually gets terminated during cool-down.
